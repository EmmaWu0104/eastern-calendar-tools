# 40 奇門遁甲 UI 接入設計

本文件是第 71 包 docs-only UI 接入設計，目標是為第 72 包前端實作做準備。第 72 包預計只做「奇門定局摘要 + 缺盤 fallback」。

本包不實作 UI、不修改 formatter / resolver / plate lookup、不修改 1080 plate JSON、不加入手動覆寫，也不加入拆補法、茅山法、無閏法或 method switch。

## 1. 目前可用能力

### 1.1 定局 formatter

可使用：

```js
resolveQimenJuFromFullTermCycleDraft(dateTimeText)
```

目前提供：

* `actualSolarTerm`
* `qimenSolarTerm`
* `status`
* `yuan`
* `dunType`
* `dunName`
* `ju`
* `hourPillar`
* `isIntercalary`
* `notes`
* `lookup`

這是後續 UI 建議使用的奇門定局來源。它已改用 cache，並已通過 regression / duplicate boundary / full range diagnostics。UI 第一版不使用 `resolveQimenJu()` 主線。

### 1.2 plate lookup

可使用：

```js
getQimenPlate({
  dunType,
  ju,
  hourPillar,
})
```

目前提供：

* `found`
* `status`
* `message`
* `meta`
* `plate`
* `lookup`

目前 plates 是 skeleton / null，大多會回 `status: "nullPlate"`。這不是錯誤，UI 應顯示定局摘要 + 缺盤 fallback。

## 2. 第 72 包 UI 接入範圍

第 72 包只做：

* 在頁面新增 / 顯示「奇門遁甲」區塊。
* 呼叫 `resolveQimenJuFromFullTermCycleDraft(...)` 取得定局結果。
* 呼叫 `getQimenPlate(...)` 取得盤面 lookup 結果。
* 顯示奇門定局摘要。
* 若 plate 為 `nullPlate` / `missingPlate` / `missingFile`，顯示缺盤 fallback。
* 若 formatter 查詢失敗，顯示奇門資料無法查詢 fallback。
* 不顯示完整九宮盤面。
* 不做手動覆寫。
* 不做 debug panel。
* 不修改 plate JSON。
* 不修改 resolver / formatter。

明確不做：

* 不做真實 1080 盤面渲染。
* 不做宮位 / 門 / 星 / 神顯示。
* 不做手動覆寫遁別 / 局數。
* 不做 method switch。
* 不做 `resolveQimenJu()` replacement。
* 不做 cache stats 顯示。
* 不做 lookup metadata 預設顯示。

## 3. 建議 UI 區塊位置

根據目前頁面既有玄學區塊設計，建議：

* 奇門遁甲區塊放在金函玉鏡日盤後面，或放在九宮飛星 / 金函玉鏡附近。
* 若目前頁面已有多個 card / section，請用同樣風格新增一個 card。
* 標題建議：`奇門遁甲`
* 副標 / 小字可選：`傳統置閏法定局`
* 第一版不需要九宮格盤面，只顯示定局摘要與 fallback。

提醒：

* 不要把奇門區塊混進金函玉鏡區塊。
* 不要共用金函玉鏡的「陽遁 / 陰遁」判斷文字，以免語意混淆。
* 奇門的遁局來自 `resolveQimenJuFromFullTermCycleDraft(...)`。

## 4. UI 資料流程

建議流程：

```text
目前查詢時間 / dateTimeText
-> resolveQimenJuFromFullTermCycleDraft(dateTimeText)
-> qimen result: dunType / dunName / ju / hourPillar / qimenSolarTerm / yuan / status / notes
-> getQimenPlate({ dunType, ju, hourPillar })
-> plate result
-> render 定局摘要
-> 若 plate.status !== "found"，render fallback message
```

formatter error 要 catch。plate lookup 不應影響 formatter 定局結果。plate lookup 的 fallback message 可直接使用 `plate.message`。

若 `plate.message` 為空但 `plate.status !== "found"`，UI 可使用預設：

```text
盤面資料尚未建立，目前僅顯示定局結果。
```

## 5. 建議顯示內容

### 5.1 正常定局摘要

第一版建議顯示：

```text
奇門遁甲
傳統置閏法定局

奇門節氣：冬至
元別：上元
遁別：陽遁
局數：一局
時柱：丙午
狀態：正常

盤面資料尚未建立，目前僅顯示定局結果。
```

### 5.2 置閏案例

```text
奇門遁甲
傳統置閏法定局

奇門節氣：大雪（置閏）
元別：上元
遁別：陰遁
局數：四局
時柱：丙午
狀態：置閏
備註：查詢時間落在 full cycle draft 置閏 timeline 內。

盤面資料尚未建立，目前僅顯示定局結果。
```

### 5.3 實際節氣與奇門節氣不同

當 `actualSolarTerm !== qimenSolarTerm` 時，建議顯示：

```text
實際節氣：冬至
奇門節氣：大雪（置閏）
狀態：置閏後接氣
```

若相同，第一版可不顯示實際節氣。若不同，顯示實際節氣有助理解超神 / 接氣 / 置閏後接氣。

### 5.4 formatter 查詢失敗

若 formatter throw，顯示：

```text
奇門遁甲資料目前無法查詢此時間。
```

不要讓整個頁面壞掉。

## 6. 欄位格式化規則

### 6.1 局數格式

`ju` 可以先轉中文：

| ju | 顯示 |
| -- | -- |
| 1 | 一局 |
| 2 | 二局 |
| 3 | 三局 |
| 4 | 四局 |
| 5 | 五局 |
| 6 | 六局 |
| 7 | 七局 |
| 8 | 八局 |
| 9 | 九局 |

若轉換失敗，可 fallback：

```text
{ju}局
```

### 6.2 置閏顯示

若 `isIntercalary === true`：

```text
奇門節氣：大雪（置閏）
```

若 false：

```text
奇門節氣：冬至
```

### 6.3 notes

若 `Array.isArray(notes) && notes.length > 0`：

* 顯示備註行。
* 多筆 notes 可用 `；` 串接。

若 notes 空：

* 不顯示備註。

### 6.4 status

status 直接顯示原字串：

* 正常
* 超神
* 接氣
* 置閏
* 置閏後接氣

不要在 UI 層重新判斷 status。

## 7. 錯誤與 fallback 規則

| 狀況 | UI 行為 |
| -- | -- |
| formatter 成功、plate `found` | 顯示定局摘要；盤面區留待未來渲染 |
| formatter 成功、plate `nullPlate` | 顯示定局摘要 + 缺盤 fallback |
| formatter 成功、plate `missingPlate` | 顯示定局摘要 + 缺盤 fallback |
| formatter 成功、plate `missingFile` | 顯示定局摘要 + 缺盤 fallback |
| formatter 成功、plate `invalidInput` | 顯示定局摘要 + 盤面查詢參數錯誤 fallback |
| formatter throw | 顯示奇門資料無法查詢 fallback；不要呼叫 plate lookup |
| plate lookup throw（理論上不應） | 顯示定局摘要 + 盤面資料讀取失敗 fallback |

第 72 包不需要建立 debug panel。console.error 是否保留由實作決定，但不要在 UI 顯示技術細節。

## 8. CSS / layout 建議

第 72 包建議：

* 沿用現有 card / section 樣式。
* 使用簡單 rows / grid 顯示欄位。
* 避免新增複雜九宮盤樣式。
* fallback message 可使用較淡的提示樣式。
* 置閏可用小 badge，但第一版可先純文字。
* 行動版不要過度擠壓。
* 不要大幅調整既有區塊 spacing。

可建議 class naming：

```text
qimen-section
qimen-summary
qimen-summary-row
qimen-label
qimen-value
qimen-note
qimen-fallback
qimen-badge
```

實際 class naming 應依現有 CSS 風格調整，不要大規模重構 CSS。

## 12. 本文件結論

第 72 包可以開始接前端，但第一版只接定局摘要與缺盤 fallback。UI 資料來源建議是 `resolveQimenJuFromFullTermCycleDraft(...)`，盤面 fallback 來源是 `getQimenPlate(...)`。

目前不渲染真實盤面，目前不做手動覆寫。formatter / plate lookup 均不回寫彼此結果。第 72 包應保持小步、低風險。
