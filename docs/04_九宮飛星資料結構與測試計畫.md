# 九宮飛星資料結構與測試計畫

## 核心模組

第一版核心位於 `src/flyingStars.js`，不接 UI。

公開函式：

- `normalizeStarNumber(number)`
- `flyStars(centerStar, direction)`
- `calculatePeriodFlyingStarChart(date)`
- `calculateAnnualFlyingStarChart(calendarResult)`
- `calculateMonthlyFlyingStarChart(calendarResult)`
- `calculateDailyFlyingStarChart(calendarResult)`
- `calculateHourlyFlyingStarChart(calendarResult)`
- `calculateAllFlyingStarCharts(calendarResult, inputDateTime)`

## 回傳資料結構

各盤回傳同一種基本結構：

```js
{
  type: "annual",
  centerStar: 1,
  direction: "forward",
  basis: {},
  palaces: {
    center: {
      id: "center",
      name: "中",
      number: 5,
      starNumber: 1,
      starName: "一白貪狼",
      starDisplayName: "① 白貪狼"
    }
  },
  sequence: ["center", "qian", "dui", "gen", "li", "kan", "kun", "zhen", "xun"],
  layout: [[...], [...], [...]]
}
```

`basis` 用來記錄該盤採用的曆法依據，例如有效年份、年柱、月支、日柱、時支、目前節氣、日盤或時盤系統名稱。

星名欄位：

- `starNumber` 保留 `1-9` 數值，不能改成圓圈字。
- `starName` 保留傳統完整星名，例如 `一白貪狼`。
- `starDisplayName` 作為 UI 顯示名稱，例如 `① 白貪狼`。
- UI 顯示中宮與九宮格星名時使用 `starDisplayName`，避免重複數字。

## 五盤整合回傳

`calculateAllFlyingStarCharts(calendarResult, inputDateTime)` 用於一次取得運盤、年盤、月盤、日盤、時盤：

```js
{
  period: calculatePeriodFlyingStarChart(inputDateTime),
  annual: calculateAnnualFlyingStarChart(calendarResult),
  monthly: calculateMonthlyFlyingStarChart(calendarResult),
  daily: calculateDailyFlyingStarChart(calendarResult),
  hourly: calculateHourlyFlyingStarChart(calendarResult)
}
```

- `period` 使用 `inputDateTime` 判斷西元年份與三元九運。
- `annual` / `monthly` / `daily` / `hourly` 使用 `calendarResult`，沿用干支曆核心已計算出的立春換年、節月、23:00 換日與時柱。
- 若 `inputDateTime` 無效，`period` 會回報明確錯誤。
- 若 `calendarResult` 缺少必要欄位，整合函式不吞錯，沿用各單盤函式的錯誤處理。

## 表格資料

- 九宮固定位置以 `PALACES` 與 `PALACE_LAYOUT` 表示。
- 順飛 / 逆飛順序以 `FLYING_DIRECTIONS` 表示。
- 九星名稱以 `STAR_NAMES` 表示。
- 月盤入中表依年支三組保存。
- 日盤依節氣六組保存。
- 時盤使用 `HOURLY_STAR_TABLES`，完整展開為 `dayBranch + hourBranch` 查表資料，不在計算時以分組公式推導。

## 測試計畫

測試資料位於 `tests/flying-stars-testcases.json`，由 `tests/run-tests.js` 執行。

已納入測試：

- 一白入中順飛。
- 九紫入中順飛。
- 一白入中逆飛。
- `2024-2043` 為九運，九入中宮。
- `2026` 年盤一白入中。
- `2026` 丙午年巳月，月盤五入中宮。
- `2026/05/29` 癸卯日，小滿，日盤七入中宮，順飛。
- `2026/05/29` 癸卯日，巳時，小滿，時盤六入中宮，順飛。
- 夏至前後日盤由順飛系統切到逆飛系統。
- 23:00 前後日柱切換後，日盤與時盤 basis 使用切換後的日柱 / 日支。

第二包新增測試：

- `calculateAllFlyingStarCharts` 於 `2026-05-29T09:30:00` 回傳五盤中心星：運 9、年 1、月 5、日 7、時 6。
- 23:00 換日整合測試，確認日盤與時盤 basis 使用切換後日柱 / 日支。
- 夏至前後整合測試，確認日盤與時盤方向由 `forward` 切到 `reverse`。
- 月盤年支三組測試：`子午卯酉`、`辰戌丑未`、`寅申巳亥`。
- 時盤查表補強：`winterForward` 與 `summerReverse`，並覆蓋三個日支分組。

第五包新增時盤完整查表核對：

- 檢查 `HOURLY_STAR_TABLES.winterForward` 與 `HOURLY_STAR_TABLES.summerReverse` 皆包含 12 個日支。
- 每個日支底下皆包含 12 個時支：子、丑、寅、卯、辰、巳、午、未、申、酉、戌、亥。
- 每個入中星值皆為 `1-9` 整數。
- 依使用者指定表逐格核對 winterForward / summerReverse，未發現漏值、轉置或錯值。

人工核對原則：

- 冬至順表分三組日支核對：`子午卯酉`、`辰戌丑未`、`寅申巳亥`。
- 夏至逆表分三組日支核對：`子午卯酉`、`辰戌丑未`、`寅申巳亥`。
- 時盤測試使用 fixture 直接指定 `dayPillar`、`hourPillar` 與節氣系統，避免被其他曆法規則干擾。

第五包新增測試案例摘要：

- winterForward：子日子時 1、子日申時 9、辰日子時 4、辰日巳時 9、寅日子時 7、寅日亥時 9、卯日巳時 6、戌日亥時 6、巳日午時 4。
- summerReverse：子日子時 9、子日申時 1、辰日子時 6、辰日巳時 1、寅日子時 3、寅日亥時 1、卯日巳時 4、戌日亥時 4、巳日午時 6。

第七包新增日盤六組節氣表核對：

- 檢查 `DAY_CENTER_SYSTEMS` 具備六組：`冬至順`、`雨水順`、`穀雨順`、`夏至逆`、`處暑逆`、`霜降逆`。
- 每組皆包含 `terms`、`direction`、`centers`。
- 方向核對：前三組為 `forward`，後三組為 `reverse`。
- 每組 `centers` 皆剛好 9 個值，且每個值皆為 `1-9` 整數。
- 24 節氣完整性核對：冬至、小寒、大寒、立春、雨水、驚蟄、春分、清明、穀雨、立夏、小滿、芒種、夏至、小暑、大暑、立秋、處暑、白露、秋分、寒露、霜降、立冬、小雪、大雪皆出現一次，且只出現一次。
- 六組 centers 依使用者指定表核對：
  - 冬至順：`[1,2,3,4,5,6,7,8,9]`
  - 雨水順：`[7,8,9,1,2,3,4,5,6]`
  - 穀雨順：`[4,5,6,7,8,9,1,2,3]`
  - 夏至逆：`[9,8,7,6,5,4,3,2,1]`
  - 處暑逆：`[3,2,1,9,8,7,6,5,4]`
  - 霜降逆：`[6,5,4,3,2,1,9,8,7]`
- 本次核對未發現漏節氣、重複節氣、錯組、順逆錯誤或 centers 錯值。

第七包新增測試案例摘要：

- 冬至順：甲子日冬至 1、壬申日立春 9，方向皆為 `forward`。
- 雨水順：甲子日雨水 7、壬申日清明 6，方向皆為 `forward`。
- 穀雨順：甲子日穀雨 4、癸卯日小滿 7、壬申日芒種 3，方向皆為 `forward`。
- 夏至逆：甲子日夏至 9、壬申日立秋 1，方向皆為 `reverse`。
- 處暑逆：甲子日處暑 3、壬申日寒露 4，方向皆為 `reverse`。
- 霜降逆：甲子日霜降 6、壬申日大雪 7，方向皆為 `reverse`。
- 測試使用 fixture 直接指定 `dayPillar` 與 `currentTerm.name`，避免被節氣資料年份、日期或日柱計算干擾。

## 測試限制

- 目前測試基準時區固定為 Asia/Taipei。
- 第一版產品邏輯仍採瀏覽器本機手錶時間，不加入真太陽時。
- 九宮飛星測試依賴既有干支曆核心輸出的 `calendarResult`。

## UI 接入

第三包已將九宮飛星五盤接入既有查詢畫面。畫面成功取得四柱 `calendarResult` 後，呼叫 `calculateAllFlyingStarCharts(calendarResult, inputDateTime)`，再顯示核心回傳結果。

UI 顯示欄位：

- 盤名：運盤、年盤、月盤、日盤、時盤。
- 中宮星：例如 `⑨ 紫右弼入中`。
- 飛法：順飛 / 逆飛。
- basis 摘要：
  - 運盤：西元年份、幾運。
  - 年盤：有效年份、年柱。
  - 月盤：年支分組、月柱、月支。
  - 日盤：日柱、目前節氣、日盤系統。
  - 時盤：日柱、時柱、目前節氣、時盤系統。
- 3x3 九宮格：固定顯示巽、離、坤 / 震、中、兌 / 艮、坎、乾，每格顯示宮名、洛書數與飛星名稱。

UI 不負責九宮飛星規則計算，只顯示 `src/flyingStars.js` 回傳結果。若九宮飛星計算失敗，四柱結果仍保留，九宮飛星區塊顯示錯誤訊息，並在 `console.error` 保留完整錯誤。

第四包補強 UI 可讀性：

- 中宮、飛法、依據分區顯示。
- basis 摘要改為多行欄位，不再塞在同一行。
- 九宮格維持固定 3x3，宮名與洛書數置上，飛星名稱置中或下方。
- 中宮格加輕微底色標示，但不加入吉凶、旺衰或斷語顏色。
- 九宮飛星區塊下方固定顯示驗證提示，提醒目前只做時間飛星、未套用真太陽時、年/月/日換盤規則。

| 輸入時間 | 驗證重點 |
