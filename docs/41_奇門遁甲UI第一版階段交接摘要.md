## 3. 目前資料流程

目前前端資料流程：

```text
dateTimeText
-> resolveQimenJuFromFullTermCycleDraft(dateTimeText)
-> qimen result
-> getQimenPlate({
     dunType: qimen.dunType,
     ju: qimen.ju,
     hourPillar: qimen.hourPillar
   })
-> render 左側定局摘要
-> render 右側 plate fallback
```

補充：

* formatter throw 時不呼叫 plate lookup。
* plate lookup 失敗不影響 formatter 定局摘要。
* plate lookup 不重新計算遁別 / 局數 / 時柱。
* plate lookup 不回寫 formatter result。
* UI 不顯示 lookup metadata。
* UI 不顯示 cache stats。

## 7. 設計邊界

* 奇門 UI 不混入金函玉鏡。
* 奇門遁局來自 `resolveQimenJuFromFullTermCycleDraft(...)`。
* plate lookup 使用 `dunType + ju + hourPillar`。
* 手動覆寫未實作；未來如果做，只影響 plate lookup effective input。
* 手動覆寫不應回寫 formatter result。
* formatter 不知道 plate 是否存在。
* plate lookup 不重新計算 formatter 欄位。
* `nullPlate` 是資料未完成，不是錯誤。
* 缺盤 fallback 不應阻止定局摘要顯示。
* UI 不預設顯示 debug metadata。
* UI 不顯示 cache stats。
