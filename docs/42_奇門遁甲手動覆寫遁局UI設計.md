# 42 奇門遁甲手動覆寫遁局 UI 設計

本文件是第 75 包 docs-only 設計文件，目標是設計未來奇門 UI 的手動覆寫遁別 / 局數功能。

本包不實作 UI、不修改 resolver、不修改 plate lookup helper、不修改 data JSON。手動覆寫只影響未來 plate lookup effective input，不回寫 formatter result，也不代表拆補法、茅山法、無閏法或 method switch。本專案第一版仍只做傳統置閏法。

## 1. 需求背景

目前奇門 UI 第一版已完成：

* 顯示 full cycle draft formatter 的自動定局結果。
* 顯示缺盤 fallback。
* layout 已調整為左側定局摘要、右側盤面區。

未來若要查盤，有時使用者可能希望人工覆寫：

* 遁別：陽遁 / 陰遁
* 局數：一局～九局

但這個覆寫用途只限於：

* 未來盤面 lookup 使用的 effective dunType / ju。
* 協助人工校對或特殊情境查表。
* 不改變自動定局結果。
* 不改變奇門置閏法 resolver。

## 2. 核心設計原則

### 2.1 自動結果永遠保留

formatter result 是自動定局結果：

* `autoDunType`
* `autoDunName`
* `autoJu`
* `hourPillar`
* `qimenSolarTerm`
* `yuan`
* `status`
* `isIntercalary`
* `notes`

手動覆寫不得修改 formatter result。

### 2.2 手動覆寫只影響 plate lookup

若使用者啟用手動覆寫：

* plate lookup 使用 manual `dunType`
* plate lookup 使用 manual `ju`
* `hourPillar` 仍使用 formatter result
* qimen summary 仍顯示自動定局
* 可另外顯示「盤面查表使用：陰遁九局」

### 2.3 不是 method switch

手動覆寫：

* 不是拆補法。
* 不是茅山法。
* 不是無閏法。
* 不是不同奇門派別切換。
* 不是 resolver replacement。
* 不是 `resolveQimenJu()` 主線切換。
* 只是 UI / plate lookup 層的查表覆寫。

### 2.4 可關閉、可還原

手動覆寫應有明確開關：

* 預設關閉。
* 關閉時使用自動定局。
* 啟用時才使用 manual effective value。
* 可一鍵恢復自動。

## 3. 建議資料模型

資料結構：

```js
{
  auto: {
    dunType: "yang",
    dunName: "陽遁",
    ju: 1,
    hourPillar: "丙午"
  },
  manual: {
    enabled: false,
    dunType: "yang",
    dunName: "陽遁",
    ju: 1
  },
  effectiveForPlateLookup: {
    dunType: "yang",
    dunName: "陽遁",
    ju: 1,
    source: "auto"
  }
}
```

啟用手動覆寫後：

```js
{
  auto: {
    dunType: "yang",
    dunName: "陽遁",
    ju: 1,
    hourPillar: "丙午"
  },
  manual: {
    enabled: true,
    dunType: "yin",
    dunName: "陰遁",
    ju: 9
  },
  effectiveForPlateLookup: {
    dunType: "yin",
    dunName: "陰遁",
    ju: 9,
    source: "manual"
  }
}
```

補充：

* `hourPillar` 不進 manual。
* `hourPillar` 永遠來自 formatter。
* `effectiveForPlateLookup` 是給 `getQimenPlate(...)` 用。
* 自動 summary 不應被 manual 覆蓋。

## 4. UI 顯示設計

### 4.1 預設狀態

預設不啟用手動覆寫。

左側定局摘要：

```text
奇門節氣：冬至
元別：上元
遁別：陽遁
局數：一局
時柱：丙午
狀態：正常
```

右側盤面區：

```text
盤面
目前使用：自動定局（陽遁一局）
盤面資料尚未建立，目前僅顯示定局結果。
```

### 4.2 啟用手動覆寫

右側盤面區可顯示：

```text
盤面
自動定局：陽遁一局
盤面查表使用：陰遁九局（手動）
盤面資料尚未建立，目前僅顯示定局結果。
```

左側定局摘要仍顯示原本自動結果，不被覆寫。

### 4.3 控制元件

建議控制元件：

* checkbox / switch：`手動覆寫盤面遁局`
* select：`遁別`
  * 陽遁
  * 陰遁
* select：`局數`
  * 一局～九局
* button / link：`恢復自動`

第一版可以放在右側盤面區，而不是左側定局摘要。

### 4.4 狀態提示

啟用 manual 時，顯示提示：

```text
手動覆寫只影響盤面查表，不改變左側自動定局。
```

## 5. plate lookup 整合流程

目前流程：

```text
qimen result
-> getQimenPlate({
     dunType: qimen.dunType,
     ju: qimen.ju,
     hourPillar: qimen.hourPillar
   })
```

未來有 manual override 後：

```text
qimen result
-> resolve effective plate input
-> getQimenPlate({
     dunType: effective.dunType,
     ju: effective.ju,
     hourPillar: qimen.hourPillar
   })
```

建議 helper：

```js
function resolveQimenPlateLookupInput(qimen, manualOverride) {
  // normalize manual state and return effective lookup input
}
```

回傳：

```js
{
  dunType: "yin",
  dunName: "陰遁",
  ju: 9,
  hourPillar: "丙午",
  source: "manual"
}
```

若未啟用 manual：

```js
{
  dunType: qimen.dunType,
  dunName: qimen.dunName,
  ju: qimen.ju,
  hourPillar: qimen.hourPillar,
  source: "auto"
}
```

## 6. 狀態管理建議

依目前 `src/main.js` 風格，可新增 module-level 狀態：

```js
let qimenManualOverride = { enabled: false, dunType: "", ju: null };
```

當查詢時間變更時有兩種策略：

| 策略 | 行為 | 優點 | 風險 |
| -- | -- | -- | -- |
| 保留 manual enabled | 查詢時間變更後仍保留 manual 選擇 | 適合連續校對同一組覆寫盤 | 使用者可能忘記目前正在手動覆寫 |
| 每次查詢重置 manual | 查詢時間變更後回到自動 | 最不容易誤用 | 連續校對時較麻煩 |

建議第一版：

* 查詢時間變更時保留 manual enabled，但 manual select 預設同步最新自動值。
* 若使用者未手動啟用，就永遠跟著自動值。
* 若已啟用 manual，保留 manual 選擇。

提醒：

* 不要讓 manual 狀態污染 formatter result。
* 不要把 manual 狀態寫入 URL 或 localStorage，第一版不需要持久化。

## 7. 錯誤與 fallback

錯誤規則：

* manual dunType 不合法：回到 auto 或顯示 invalidInput。
* manual ju 不合法：回到 auto 或顯示 invalidInput。
* 建議第一版在 UI 控制元件限制合法值，所以不應出現非法 manual。
* 若 effective input 仍造成 `getQimenPlate(...)` 回 `invalidInput`：
  * 顯示 `奇門盤面查詢參數不完整，暫時無法顯示盤面。`
  * 左側定局摘要仍保留。

## 11. 本文件結論

手動覆寫是 UI / plate lookup 層功能。手動覆寫只改變 `getQimenPlate(...)` 的 effective dunType / ju，不修改 formatter result，也不代表 method switch。

左側自動定局摘要應永遠保留。右側盤面區可顯示自動 vs 手動查表使用值。下一步若要實作，應保持小包，只做控制元件與 effective lookup input，不做真實盤面。
