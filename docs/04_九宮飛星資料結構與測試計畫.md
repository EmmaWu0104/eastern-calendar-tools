# 九宮飛星資料結構與測試計畫

## 核心模組

第一版核心位於 `src/flyingStars.js`，不接 UI。

公開函式：

- `normalizeStarNumber(number)`
- `flyStars(centerStar, direction)`
- `calculatePeriodFlyingStarChart(date)`
- `calculateAnnualFlyingStarChart(calendarResult)`
- `calculateMonthlyFlyingStarChart(calendarResult)`
- `calculateDailyFlyingStarChart(calendarResult)`
- `calculateHourlyFlyingStarChart(calendarResult)`
- `calculateAllFlyingStarCharts(calendarResult, inputDateTime)`

## 回傳資料結構

各盤回傳同一種基本結構：

```js
{
  type: "annual",
  centerStar: 1,
  direction: "forward",
  basis: {},
  palaces: {
    center: { id: "center", name: "中", number: 5, starNumber: 1, starName: "一白貪狼" }
  },
  sequence: ["center", "qian", "dui", "gen", "li", "kan", "kun", "zhen", "xun"],
  layout: [[...], [...], [...]]
}
```

`basis` 用來記錄該盤採用的曆法依據，例如有效年份、年柱、月支、日柱、時支、目前節氣、日盤或時盤系統名稱。

## 五盤整合回傳

`calculateAllFlyingStarCharts(calendarResult, inputDateTime)` 用於一次取得運盤、年盤、月盤、日盤、時盤：

```js
{
  period: calculatePeriodFlyingStarChart(inputDateTime),
  annual: calculateAnnualFlyingStarChart(calendarResult),
  monthly: calculateMonthlyFlyingStarChart(calendarResult),
  daily: calculateDailyFlyingStarChart(calendarResult),
  hourly: calculateHourlyFlyingStarChart(calendarResult)
}
```

- `period` 使用 `inputDateTime` 判斷西元年份與三元九運。
- `annual` / `monthly` / `daily` / `hourly` 使用 `calendarResult`，沿用干支曆核心已計算出的立春換年、節月、23:00 換日與時柱。
- 若 `inputDateTime` 無效，`period` 會回報明確錯誤。
- 若 `calendarResult` 缺少必要欄位，整合函式不吞錯，沿用各單盤函式的錯誤處理。

## 表格資料

- 九宮固定位置以 `PALACES` 與 `PALACE_LAYOUT` 表示。
- 順飛 / 逆飛順序以 `FLYING_DIRECTIONS` 表示。
- 九星名稱以 `STAR_NAMES` 表示。
- 月盤入中表依年支三組保存。
- 日盤依節氣六組保存。
- 時盤使用 `HOURLY_STAR_TABLES`，完整展開為 `dayBranch + hourBranch` 查表資料，不在計算時以分組公式推導。

## 測試計畫

測試資料位於 `tests/flying-stars-testcases.json`，由 `tests/run-tests.js` 執行。

已納入測試：

- 一白入中順飛。
- 九紫入中順飛。
- 一白入中逆飛。
- `2024-2043` 為九運，九入中宮。
- `2026` 年盤一白入中。
- `2026` 丙午年巳月，月盤五入中宮。
- `2026/05/29` 癸卯日，小滿，日盤七入中宮，順飛。
- `2026/05/29` 癸卯日，巳時，小滿，時盤六入中宮，順飛。
- 夏至前後日盤由順飛系統切到逆飛系統。
- 23:00 前後日柱切換後，日盤與時盤 basis 使用切換後的日柱 / 日支。

第二包新增測試：

- `calculateAllFlyingStarCharts` 於 `2026-05-29T09:30:00` 回傳五盤中心星：運 9、年 1、月 5、日 7、時 6。
- 23:00 換日整合測試，確認日盤與時盤 basis 使用切換後日柱 / 日支。
- 夏至前後整合測試，確認日盤與時盤方向由 `forward` 切到 `reverse`。
- 月盤年支三組測試：`子午卯酉`、`辰戌丑未`、`寅申巳亥`。
- 時盤查表補強：`winterForward` 與 `summerReverse`，並覆蓋三個日支分組。

## 測試限制

- 目前測試基準時區固定為 Asia/Taipei。
- 第一版產品邏輯仍採瀏覽器本機手錶時間，不加入真太陽時。
- 九宮飛星測試依賴既有干支曆核心輸出的 `calendarResult`。

## UI 接入

第三包已將九宮飛星五盤接入既有查詢畫面。畫面成功取得四柱 `calendarResult` 後，呼叫 `calculateAllFlyingStarCharts(calendarResult, inputDateTime)`，再顯示核心回傳結果。

UI 顯示欄位：

- 盤名：運盤、年盤、月盤、日盤、時盤。
- 中宮星：例如 `九紫右弼入中`。
- 飛法：順飛 / 逆飛。
- basis 摘要：
  - 運盤：西元年份、幾運。
  - 年盤：有效年份、年柱。
  - 月盤：年支分組、月柱、月支。
  - 日盤：日柱、目前節氣、日盤系統。
  - 時盤：日柱、時柱、目前節氣、時盤系統。
- 3x3 九宮格：固定顯示巽、離、坤 / 震、中、兌 / 艮、坎、乾，每格顯示宮名、洛書數與飛星名稱。

UI 不負責九宮飛星規則計算，只顯示 `src/flyingStars.js` 回傳結果。若九宮飛星計算失敗，四柱結果仍保留，九宮飛星區塊顯示錯誤訊息，並在 `console.error` 保留完整錯誤。
