# 金函玉鏡日盤階段交接摘要

## 一、目前完成狀態

金函玉鏡日盤目前已完成第一階段可用版本：

- 120 盤資料整理完成。
- 陽遁 / 陰遁各 60 盤。
- 六十甲子日皆有青龍黑黃道十二時辰資料。
- 已有查表模組。
- 已有 UI 顯示。
- 已有超神接氣 v1 自動判斷。
- UI 預設自動套用陰陽遁。
- 右上角保留手動覆寫選單。
- 已新增現在時間按鈕。
- 已顯示查詢日星期。
- 已標記青龍黑黃道目前所在時辰。
- 左側摘要已精簡為日柱、金函玉鏡盤、中宮三行。

## 二、資料檔摘要

`data/jinhan_yujing_day_pan.json` 目前包含：

- 60 個日柱。
- 每個日柱包含：
  - 陽遁盤。
  - 陰遁盤。
  - `blackYellowHours`。
- 每盤包含：
  - `meta`。
  - `palaces`。
- `meta` 至少包含：
  - `pillar`
  - `dunType`
  - `label`
  - `center`
  - `xishen`
  - `caishen`
  - `yinGuishen`
  - `yangGuishen`
- `palaces` 包含八宮：
  - 坎、艮、震、巽、離、坤、兌、乾
- `blackYellowHours` 為 12 筆，對應子、丑、寅、卯、辰、巳、午、未、申、酉、戌、亥時。

本階段不修改節氣資料，也不修改 `data/solar_terms_1899_2101.json`。

## 三、查表模組摘要

`src/jinhanYujing.js` 目前提供：

- `getJinhanYujingDayPan(dayPillar, dunType)`
- `getJinhanBlackYellowHours(dayPillar)`
- `getJinhanDeitiesByPalace(meta)`

函式用途：

- `getJinhanYujingDayPan(dayPillar, dunType)`：依日柱 + 陰陽遁查金函玉鏡日盤。
- `getJinhanBlackYellowHours(dayPillar)`：依日柱查青龍黑黃道十二時辰。
- `getJinhanDeitiesByPalace(meta)`：依 `meta` 的喜神、財神、陰貴神、陽貴神整理各宮 chip。

## 四、超神接氣 v1 規則摘要

`src/jinhanDunType.js` 目前已實作：

```js
getJinhanDunType(dateTime, calendarResult, solarTerms)
```

v1 規則：

- 換遁日一定是甲日。
- 地支不計。
- 冬至換入陽遁。
- 夏至換入陰遁。
- 節氣交節日干為甲：正授，當日換遁。
- 節氣交節日干為乙、丙、丁、戊：接氣，往前推到最近甲日換遁。
- 節氣交節日干為己、庚、辛、壬、癸：超神，往後推到最近甲日換遁。
- 節氣日干依節氣精確交節時間判斷。
- 節氣交節時間若落在 23:00 後，依本工具既有 23:00 換日規則，使用新干支日。
- 不使用民用日期 00:00 的日干判斷。
- 不使用真太陽時。

## 五、年度區間判斷摘要

年度區間判斷：

- 查詢日介於上一年冬至換遁日與今年夏至換遁日前：陽遁。
- 查詢日介於今年夏至換遁日與今年冬至換遁日前：陰遁。
- 查詢日大於等於今年冬至換遁日：陽遁。
- 一月日期不可錯用今年冬至，需參考上一年冬至換遁日。

## 六、UI 行為摘要

目前 UI 行為：

- 預設自動呼叫 `getJinhanDunType(...)`。
- 若回傳 `resolved + dunType`，自動使用該陰陽遁查盤。
- 右上角 select 保留，可以手動覆寫。
- 使用者手動覆寫後，不會被自動判斷立即覆蓋。
- 重新查詢日期後恢復自動判斷優先。
- 自動判斷失敗時 fallback 到 select 目前值。
- 左側摘要只顯示：
  - 日柱
  - 金函玉鏡盤
  - 中宮
- 不顯示詳細自動判斷理由，以維持畫面整潔。
- 詳細判斷規則以文件與測試保存。

## 七、查詢時間 UI 摘要

第 13 包完成下列查詢時間功能：

- 日期時間區有「現在時間」按鈕。
- 點擊後填入瀏覽器本機目前時間，並立即重新查詢。
- 顯示查詢日星期，例如「查詢日：星期一」。
- 青龍黑黃道表格上方顯示目前時辰，例如「目前時辰：巳時（09 ~ 11）」。
- 青龍黑黃道表格中目前所在時辰列會標記：
  - 淡色底。
  - 左側提示線。
  - 第一欄顯示「▶ 目前」。
- 目前時辰列比對同時支援：
  - `hour.index`。
  - 12 列順序 fallback。

目前測試涵蓋：

- 九宮飛星案例。
- 日干吉神案例。
- 納音案例。
- 金函玉鏡資料完整性檢查：

  - 夏至正授。
  - 夏至接氣。

  - 跨年一月。
  - 23:00 後交節使用新干支日。

## 九、目前限制

- 未實作真太陽時。
- 未實作地點 / 經度校正。

- UI 目前不顯示超神接氣詳細 reason；詳細規則保存在文件與測試中。
- 不修改 `data/solar_terms_1899_2101.json`。
- 不混入其他流派規則。
