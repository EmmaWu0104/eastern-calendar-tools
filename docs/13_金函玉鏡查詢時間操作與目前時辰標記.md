# 金函玉鏡查詢時間操作與目前時辰標記

## 一、功能範圍

第 13 包新增查詢時間操作與黑黃道目前時辰標記：

- 在日期時間查詢區新增「現在時間」按鈕。
- 顯示目前查詢時間是星期幾。
- 青龍黑黃道依傳入時間標記目前所在時辰。
- 青龍黑黃道表格上方顯示目前時辰提示。

本包不修改金函玉鏡資料、不修改節氣資料、不修改超神接氣 v1 規則。

## 二、金函玉鏡摘要

金函玉鏡左側摘要只保留三行：

- 日柱
- 金函玉鏡盤
- 中宮

自動陰陽遁判斷仍會影響盤面，但不在摘要區顯示詳細判斷理由。右上角陰陽遁選單仍保留，可手動覆寫。

## 三、現在時間按鈕

「現在時間」按鈕會：

- 將 `datetime-local` 設為瀏覽器本機目前時間。
- 格式沿用既有 `YYYY-MM-DDTHH:mm`。
- 設定後立即重新查詢。
- 不加入秒數。
- 不套用真太陽時。

## 四、星期顯示

查詢成功後，日期時間查詢區會顯示：

```text
查詢日：星期一
```

星期依 `datetime-local` 的輸入值，以瀏覽器本機時間解析，不使用 UTC，避免跨時區造成星期偏移。

若輸入空值或格式無法解析，顯示 `--`。

## 五、青龍黑黃道目前時辰標記

青龍黑黃道表格會依傳入時間的小時，標記目前所在時辰列。

表格上方會顯示目前時辰，例如：

```text
目前時辰：巳時（09 ~ 11）
```

目前列使用：

- 淡色底色。
- 左側提示線。
- 時段前顯示 `▶ 目前` 標記。

不新增欄位，避免表格過度擁擠。

## 六、時辰 index 判斷

時辰 index 對應黑黃道資料的 `hour.index`，範圍為 1 到 12。

判斷規則：

| 時辰 | 小時 | index |
| --- | --- | ---: |
| 子 | 23, 0 | 1 |
| 丑 | 1, 2 | 2 |
| 寅 | 3, 4 | 3 |
| 卯 | 5, 6 | 4 |
| 辰 | 7, 8 | 5 |
| 巳 | 9, 10 | 6 |
| 午 | 11, 12 | 7 |
| 未 | 13, 14 | 8 |
| 申 | 15, 16 | 9 |
| 酉 | 17, 18 | 10 |
| 戌 | 19, 20 | 11 |
| 亥 | 21, 22 | 12 |

此處只用於青龍黑黃道列標記，不改變既有干支曆 23:00 換日規則。

## 七、列標記比對

目前列標記同時支援兩種比對：

- 優先使用資料列的 `hour.index` 與目前時辰 index 比對。
- 若資料 index 型別或內容未命中，使用表格列順序作為 fallback，也就是第 1 列對應子時、第 2 列對應丑時，直到第 12 列對應亥時。

因此即使資料中的 `hour.index` 未來改為字串或缺漏，只要黑黃道 12 列順序維持不變，畫面仍能標記目前時辰列。

## 八、Console 404 處理

瀏覽器可能會自動請求 `/favicon.ico`。本專案目前不需要實體 favicon 檔，因此 `index.html` 使用空 favicon data URI：

```html
<link rel="icon" href="data:,">
```

這可避免瀏覽器自動請求 `/favicon.ico` 造成 Console 404。若未來 Console 出現其他 404，需另外檢查實際缺少的 JS / CSS / JSON 路徑，不應用 favicon 規避方式掩蓋真正資源錯誤。

## 九、限制

- 不修改 `data/jinhan_yujing_day_pan.json`。
- 不修改 `data/solar_terms_1899_2101.json`。
- 不修改 `src/jinhanYujing.js`。
- 不修改 `src/jinhanDunType.js`。
- 不修改超神接氣 v1 規則。
- 不實作真太陽時。
