# 金函玉鏡超神接氣 v1 實作規則

## 一、版本定位

本文件記錄金函玉鏡超神接氣 v1 已實作的工程規則。

v1 依使用者確認規則實作，只處理陰陽遁判斷，不修改金函玉鏡 120 盤資料，不修改節氣資料，也不加入真太陽時。

本輪 UI 仍不自動套用判斷結果，畫面保留手動選擇「陽遁 / 陰遁」。

## 二、v1 採用規則

- 換遁日一定是甲日。
- 地支不計。
- 冬至換入陽遁。
- 夏至換入陰遁。
- 冬至 / 夏至當日天干若為甲，稱正授，當日換遁。
- 冬至 / 夏至當日天干若為乙、丙、丁、戊，往前推到最近的甲日換遁，稱接氣。
- 冬至 / 夏至當日天干若為己、庚、辛、壬、癸，往後推到最近的甲日換遁，稱超神。

## 三、換遁 offset 表

| 節氣日干 | 換遁日 | 模式 |
| --- | ---: | --- |
| 甲 | 當日 | 正授 |
| 乙 | 前 1 日甲日 | 接氣 |
| 丙 | 前 2 日甲日 | 接氣 |
| 丁 | 前 3 日甲日 | 接氣 |
| 戊 | 前 4 日甲日 | 接氣 |
| 己 | 後 5 日甲日 | 超神 |
| 庚 | 後 4 日甲日 | 超神 |
| 辛 | 後 3 日甲日 | 超神 |
| 壬 | 後 2 日甲日 | 超神 |
| 癸 | 後 1 日甲日 | 超神 |

## 四、節氣日干判斷

冬至 / 夏至當日天干不使用民用日期 00:00 的日干。

v1 使用節氣精確交節時間，並依本工具既有干支曆核心與 23:00 換日規則，計算交節時刻所屬日柱，再取該日柱天干。

因此若冬至 / 夏至交節時間落在 23:00 之後，會以 23:00 後的新干支日作為該節氣所屬干支日。

例如交節時間為 `2025-12-21T23:02:48+08:00` 時，不以 `2025-12-21 00:00` 的日干判斷，而以 23:00 換日後的新干支日判斷。

## 五、年度判斷

v1 不只看查詢日期當年度冬至。

判斷時會計算：

- 上一年冬至換遁日
- 今年夏至換遁日
- 今年冬至換遁日

基本區間：

- 查詢日 >= 上一年冬至換遁日，且 < 今年夏至換遁日：陽遁。
- 查詢日 >= 今年夏至換遁日，且 < 今年冬至換遁日：陰遁。
- 查詢日 >= 今年冬至換遁日：陽遁。

若查詢日早於上一年冬至換遁日，會再補前一年夏至與前二年冬至資料判斷。

## 六、超神與接氣期間

接氣時，換遁日早於節氣交節日。查詢日若已到前方甲日，即使用節氣後理論方向：

- 冬至接氣：使用陽遁。
- 夏至接氣：使用陰遁。

超神時，換遁日晚於節氣交節日。查詢日若已到節氣日但尚未到後方甲日，仍使用換遁前方向：

- 冬至超神、尚未到後方甲日：仍用陰遁。
- 夏至超神、尚未到後方甲日：仍用陽遁。

到後方甲日後，再切入：

- 冬至超神後方甲日：陽遁。
- 夏至超神後方甲日：陰遁。

## 七、回傳格式

`getJinhanDunType(dateTime, calendarResult, solarTerms)` 成功判斷時回傳：

```js
{
  status: "resolved",
  dunType: "陽遁",
  mode: "接氣",
  boundary: "冬至",
  reason: "依金函玉鏡超神接氣 v1：冬至交節日干為丙，接氣，已於前方甲日換入陽遁。"
}
```

若缺少節氣資料或無法判斷，回傳：

```js
{
  status: "unsupported",
  dunType: null,
  mode: "unknown",
  boundary: null,
  reason: "缺少節氣資料，無法判斷金函玉鏡陰陽遁。"
}
```

## 八、UI 狀態

第 11 包只實作判斷函式與測試。

UI 目前仍保留：

- 手動選擇陽遁 / 陰遁。
- 不使用 `dunTypeStatus.dunType` 自動覆蓋手動選擇。
- 不自動切換金函玉鏡盤。

自動 / 手動模式切換、判斷理由展示優化，留待後續包處理。

## 九、測試重點

v1 測試包含：

- 缺少 `solarTerms` 回傳 `unsupported`。
- 冬至正授。
- 冬至接氣。
- 冬至超神，後方甲日前仍用陰遁。
- 冬至超神，後方甲日後改用陽遁。
- 夏至正授。
- 夏至接氣。
- 夏至超神，後方甲日前仍用陽遁。
- 夏至超神，後方甲日後改用陰遁。
- 跨年一月使用上一年冬至換遁日。
- 23:00 後交節使用新干支日天干，不使用民用日期 00:00 日干。
