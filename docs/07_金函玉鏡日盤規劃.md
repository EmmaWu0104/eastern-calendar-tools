# 金函玉鏡日盤規劃

本文件只記錄金函玉鏡日盤第一版的規劃，不代表已實作。規則尚未確認的部分只列為待確認，不在文件中硬寫演算法。

## 功能背景

目前專案已完成：

- 24 節氣 JSON。
- 干支曆核心。
- 四柱查詢。
- 納音五行。
- 日干吉神九宮格。
- 九宮飛星：運盤、年盤、月盤、日盤、時盤。

金函玉鏡日盤第一版沿用既有時間規則：

- 使用手錶時間 / 瀏覽器本機時間。
- 不套用真太陽時。
- 立春換年。
- 12 節換月。
- 23:00 起換日。
- 不修改 `data/solar_terms_1899_2101.json`。

## 核心設計

金函玉鏡不是單純 60 甲子日查表，而是：

- 60 甲子日 × 陽遁。
- 60 甲子日 × 陰遁。

合計 120 盤。因此查詢 key 應包含：

- `dayPillar`，例如 `甲子`。
- `dunType`，例如 `陽遁` 或 `陰遁`。

建議 lookup function 名稱先規劃為：

```js
getJinhanYujingDayPan(dayPillar, dunType)
```

不要只用：

```js
getJinhanYujingDayPanByPillar(dayPillar)
```

原因是同一個日柱會有陽遁盤與陰遁盤兩種結果。

## 喜財陰陽規劃

第一版畫面上建議合併顯示在金函玉鏡九宮格內。

資料概念上應標示為可獨立：

- 金函玉鏡本體：陰陽遁、日柱、中宮、八門、九星。
- 方位神資料：喜神、財神、陰貴神、陽貴神。

UI 顯示方式：

- 喜神顯示 `喜`。
- 財神顯示 `財`。
- 陰貴神顯示 `陰`。
- 陽貴神顯示 `陽`。

若多個神落在同一宮，要並列顯示，不可覆蓋或遺失。

## 黑黃道十二神時辰表規劃

目前資料中每個日柱另有「黑黃道」表，包含 12 個時辰資料，例如：

- 時辰序號。
- 時柱。
- 時段，例如 `23 ~ 01`。
- 黑黃道十二神，例如金匱、天德、白虎、玉堂等。
- 附註，例如天乙貴人、五不遇時、截路空亡。

第一版建議不要塞進九宮格，應規劃為金函玉鏡盤下方的獨立表格區：

```text
今日十二時辰黑黃道
```

欄位建議：

- 時段。
- 時柱。
- 黑黃道神。
- 吉凶標示。
- 附註。

## 超神接氣換遁規則

金函玉鏡陰陽遁不可直接用普通冬至 / 夏至切換。

原文規則：

> 如遇冬至、夏至，以甲日換順逆，如甲在冬至前七日，仍依夏至後逆行，冬至後三日亦必逆行，須到第四逢之甲，方用冬至後順行之例，名曰「超神」，謂中氣已到，六甲一旬十日，超過中氣也。
> 如甲子日在冬至前四日，即依冬至後順行，名曰「接氣」，謂中氣未到，六甲旬頭先到，迎接中氣也。

規則草案：

- 冬至、夏至附近不是以節氣時間當下直接換遁。
- 換遁需要參照甲日或六甲旬頭與中氣的相對位置。
- 「超神」描述中氣已到，但仍依前一系統行到指定甲日。
- 「接氣」描述中氣未到，但旬頭先到，提前採用後一系統。
- 在規則未完全確認前，不可自行硬寫演算法。

待確認事項：

- 「第四逢之甲」的精確程式判斷方式。
- 冬至邊界的完整案例。
- 夏至邊界是否完全對稱。
- 甲日是否限甲子，或所有六甲旬頭皆需納入。
- 超神 / 接氣與既有 23:00 換日規則如何銜接。
- 節氣時間使用 `data/solar_terms_1899_2101.json`，但不得修改該資料檔。

建議未來函式名稱：

```js
getJinhanDunType(dateTime, calendarResult, solarTerms)
```

回傳規劃：

```json
{
  "dunType": "陽遁",
  "mode": "接氣",
  "boundary": "冬至",
  "reason": "甲子日在冬至前四日，依冬至後順行"
}
```

或：

```json
{
  "dunType": "陰遁",
  "mode": "超神",
  "boundary": "冬至",
  "reason": "冬至已到，但仍未至換遁甲日，仍依夏至後逆行"
}
```

在規則未完全確認前，UI 或設定可先支援：

- 自動判斷：待確認。
- 手動選擇：陽遁 / 陰遁。

## 測試案例規劃

資料完整性測試：

- 應有 60 個日柱。
- 每個日柱應有陽遁與陰遁。
- 每一盤應有 8 宮：坎、艮、震、巽、離、坤、兌、乾。
- 每宮應有 `door` 與 `star`。
- 每盤應有 `center`。
- 每個日柱應有黑黃道 12 時辰資料。

查表測試：

- `甲子 + 陽遁` => `陽遁甲子日`。
- `甲子 + 陰遁` => `陰遁甲子日`。
- `乙丑 + 陽遁` => `陽遁乙丑日`。
- `乙丑 + 陰遁` => `陰遁乙丑日`。
- 無效日柱不可 throw。
- 無效 `dunType` 不可 throw。

喜財陰陽測試：

- 同宮多神不可遺失。
- 甲子日艮宮同時有喜、財、陰時，應可並列顯示。
- 陽貴神若在坤宮，應顯示於坤宮。

黑黃道測試：

- 每日 12 筆。
- 時段由 `23~01` 起，至 `21~23` 止。
- 可保留 emoji，但資料層最好能拆出 `type`，例如 `yellow` / `black` 或 `auspicious` / `inauspicious`。

超神接氣測試：

先列為待確認，不可寫死。待確認案例包含：

- 甲日在冬至前七日。
- 冬至後三日仍逆行。
- 到第四逢之甲才用冬至後順行。
- 甲子日在冬至前四日提前順行。
- 夏至邊界對應規則。
