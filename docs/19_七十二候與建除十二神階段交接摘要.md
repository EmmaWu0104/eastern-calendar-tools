# 七十二候與建除十二神階段交接摘要

## 一、本階段完成範圍

本階段完成七十二候與建除十二神第一版可用功能。

七十二候已完成：

- 七十二候靜態資料。
- 七十二候查表與區間判斷。
- 干支曆查詢結果整合。
- UI 顯示目前七十二候、候區間與下一候。

建除十二神已完成：

- 建除十二神規則文件。
- 建除十二神純計算模組。
- 干支曆查詢結果整合。
- UI 顯示於四柱區塊的日柱卡片下方。

## 二、七十二候完成狀態

七十二候資料檔為：

- `data/seventy_two_hou.json`

七十二候核心模組為：

- `src/seventyTwoHou.js`

目前 `calculateBaziFromSolarTerms(...)` 已回傳：

- `currentHou`
- `nextHou`

UI 顯示位置：

- 「節氣與月令」區塊。

目前 UI 顯示：

- 目前七十二候。
- 候區間。
- 下一候。

七十二候區間規則：

- 沿用既有 24 節氣時間。
- 以目前節氣交節時間到下一節氣交節時間為一個區間。
- 將節氣區間三等分為初候、次候、末候。
- 區間採左閉右開 `[start, end)`。
- 不受 23:00 換日影響。
- 不使用真太陽時。
- 不做太陽黃經 5 度精算。

## 三、建除十二神完成狀態

建除十二神規則文件為：

- `docs/16_建除十二神規則與實作規劃.md`

建除十二神核心模組為：

- `src/jianchu.js`

目前 `calculateBaziFromSolarTerms(...)` 已回傳：

- `jianchu`

UI 顯示位置：

- 四柱區塊的「日柱」卡片下方。

UI 顯示格式：

```text
建除：破日
```

建除十二神第一版規則：

- 依月令月支與日柱地支計算。
- 日支沿用本工具 23:00 換日規則。
- 月支沿用 12 節切換規則。
- 不查每日通勝表。
- 不做宜忌、吉凶、旺衰或斷語。

## 四、主要修改檔案摘要

- `data/seventy_two_hou.json`：七十二候靜態資料。
- `src/seventyTwoHou.js`：七十二候查表、目前候與下一候判斷。
- `src/jianchu.js`：建除十二神純計算模組。
- `src/bazi.js`：整合 `currentHou`、`nextHou`、`jianchu` 到干支曆查詢結果。
- `src/main.js`：顯示七十二候、下一候與日柱卡片下方的建除十二神。
- `styles/main.css`：七十二候與建除十二神顯示樣式。
- `tests/run-tests.js`：七十二候與建除十二神測試。
- `docs/15_七十二候資料設計與實作規劃.md`：七十二候資料與實作規劃。
- `docs/16_建除十二神規則與實作規劃.md`：建除十二神規則與實作規劃。

## 五、測試摘要

目前測試涵蓋：

- 七十二候資料完整性。
- 七十二候初候、次候、末候邊界。
- 下一候同節氣與跨節氣。
- 大寒末候跨年到次年立春初候。
- 建除十二神純函式測試。
- 建除十二神整合測試。
- 23:00 換日後建除跟著日柱地支切換。
- 既有干支曆、九宮飛星、金函玉鏡、納音測試皆維持通過。

## 六、目前限制

- 未使用真太陽時。
- 七十二候不是太陽黃經 5 度精算。
- 建除十二神不是每日通勝表查表。
- 建除十二神不含宜忌與吉凶斷語。
- 日柱基準日仍依目前工具規則，後續可再交叉驗證。

## 七、後續可選工作

- 補真太陽時進階功能。
- 加入更多每日神煞或擇日標記。
- 建除十二神加入說明文字，但不要在第一版加入宜忌斷語。
- 整理 GitHub README 功能說明。
- 若要對照通勝，需另開校驗資料與案例。

## 八、本階段結論

七十二候與建除十二神已完成第一版可用功能，包含資料、計算、干支曆結果整合、UI 顯示與測試覆蓋。本階段未修改金函玉鏡日盤、九宮飛星，亦未改動七十二候以外的節氣核心規則，可作為下一階段功能擴充基礎。
