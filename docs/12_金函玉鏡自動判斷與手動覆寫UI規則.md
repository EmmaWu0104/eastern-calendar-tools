# 金函玉鏡自動判斷與手動覆寫 UI 規則

## 一、目的

本文件記錄金函玉鏡日盤 UI 在超神接氣 v1 判斷完成後的顯示與操作規則。

UI 預設自動套用 `getJinhanDunType(...)` 回傳的陰陽遁，但仍保留右上角「陽遁 / 陰遁」選單作為人工覆寫。

## 二、預設自動套用

每次完成日期時間查詢後：

- 重新載入或取得 normalize 後的節氣資料。
- 呼叫 `getJinhanDunType(dateTime, calendarResult, solarTerms)`。
- 若回傳 `status === "resolved"` 且 `dunType` 有值，預設使用該 `dunType` 查金函玉鏡日盤。
- 若使用者尚未人工覆寫，右上角 select 會同步顯示自動判斷結果。

## 三、手動覆寫

右上角 select 保留為人工覆寫入口。

使用者手動切換 select 後：

- 視為手動覆寫。
- 金函玉鏡日盤改用 select 值查盤。
- 不再被自動判斷立即覆蓋。
- 摘要顯示「陰陽遁來源：手動覆寫」。
- 若自動判斷有建議值，摘要顯示「自動建議」。

## 四、重新計算日期

使用者重新按「查詢」或在日期時間欄位按 Enter 完成新查詢後：

- 清除上一輪手動覆寫狀態。
- 恢復自動判斷優先。
- 若新日期自動判斷成功，select 會同步為自動判斷結果。

## 五、自動判斷失敗

若 `getJinhanDunType(...)` 回傳 `unsupported` 或未提供有效 `dunType`：

- 不清空金函玉鏡日盤。
- 不自動切換 select。
- 使用 select 目前值查盤。
- 摘要顯示「陰陽遁來源：手動選擇」。
- 摘要顯示判斷失敗原因。

## 六、摘要顯示

自動成功且未手動覆寫時，摘要顯示：

- 陰陽遁來源：自動判斷
- 目前使用：陽遁 / 陰遁
- 自動判斷：resolved
- 換遁模式：正授 / 接氣 / 超神
- 換遁邊界：冬至 / 夏至
- 判斷理由

使用者手動覆寫時，摘要顯示：

- 陰陽遁來源：手動覆寫
- 目前使用：陽遁 / 陰遁
- 自動判斷：resolved 或 unsupported
- 自動建議：陽遁 / 陰遁（若有）
- 換遁模式、換遁邊界與判斷理由（若有）

自動判斷失敗時，摘要顯示：

- 陰陽遁來源：手動選擇
- 目前使用：select 目前值
- 自動判斷：unsupported
- 判斷理由

## 七、限制

- UI 不修改金函玉鏡 120 盤資料。
- UI 不修改 `data/solar_terms_1899_2101.json`。
- UI 不修改超神接氣 v1 規則。
- UI 不加入其他流派規則。
- UI 不加入真太陽時。
- UI 不移除手動選擇陰陽遁。
