# 每日資訊階段交接摘要

## 一、目前完成狀態

每日資訊第一版已完成可用流程，包含：

- 規則文件。
- 純函式模組。
- `bazi` 查詢結果整合。
- UI 顯示。
- emoji 圖標顯示。
- 測試與人工驗收。

## 二、主要完成檔案

- `docs/20_每日資訊規則與實作規劃.md`：每日資訊規則、UI 圖標規格與後續分包規劃。
- `src/dailyInfo.js`：每日資訊純函式模組，維持純文字資料回傳。
- `src/bazi.js`：整合 `dailyInfo` 到 `calculateBaziFromSolarTerms(...)` 回傳結果。
- `src/main.js`：將衣著資訊與日柱卡片每日資訊顯示到 UI。
- `styles/main.css`：每日資訊 UI 樣式。
- `tests/run-tests.js`：每日資訊純函式測試與 `bazi` 整合測試。

## 三、每日資訊項目摘要

第一版包含：

1. 衣著顏色。
2. 衝煞。
3. 歲破日。
4. 四離四絕日。
5. 天赦日。
6. 三伏日。

## 四、時間規則摘要

- 年柱沿用立春換年。
- 日柱沿用 23:00 換日。
- 每日資訊使用與日柱一致的有效日。
- 四離四絕的「前一日」使用 23:00 有效日，不是民用 00:00 日期。
- 節氣季節依目前節氣 `currentTerm` 判斷。
- 三伏依夏至、立秋有效日與庚日序推算。
- 第一版不使用真太陽時。

## 五、資料與顯示分離摘要

- `src/dailyInfo.js` 回傳純文字資料。
- emoji 不寫入 `dailyInfo` 核心計算結果。
- emoji 由 `src/main.js` UI helper 組合。
- 這樣可避免影響測試、資料匯出與未來純文字用途。

## 六、UI 顯示摘要

衣著顯示於查詢區「查詢日」下方：

```text
衣著：
🧥 讚的 ...
🧥 吉的 ...
⛔ 大忌 ...
```

日柱卡片顯示於建除下方：

```text
❌ 衝煞：...
💀 歲破日
😇 天赦日
♨ 初伏 / 中伏 / 末伏
💀 離日 / 絕日
```

顯示規則：

- 非歲破日不顯示歲破行。
- 非天赦日不顯示天赦行。
- 非三伏日不顯示三伏行。
- 非四離四絕日不顯示離日 / 絕日行。
- 不顯示「否」。
- 不顯示空白行。

## 七、測試摘要

已完成測試：

- `src/dailyInfo.js` 純函式測試。
- `bazi` dailyInfo 整合測試。
- 衣著、衝煞、歲破、天赦、四離四絕、三伏。
- 22:59 / 23:00 換日一致性。
- `npm test` 通過。

## 九、目前限制

- 不加入其他神煞。
- 不加入宜忌斷語。
- 不加入吉凶斷語。
- 三伏第一版採目前規則，後續若要對照通勝需另開校驗案例。
- 不使用真太陽時。
- 尚未做離線版打包。
- 手機 `file://` 離線開啟暫不處理。
- GitHub Pages 部署可於日盤資訊完成後再處理。
