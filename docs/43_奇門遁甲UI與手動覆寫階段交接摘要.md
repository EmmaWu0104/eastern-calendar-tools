## 3. 目前資料流程

```text
dateTimeText
-> resolveQimenJuFromFullTermCycleDraft(dateTimeText)
-> qimen auto result
-> syncQimenManualControlsWithAuto(qimen)
-> resolveQimenPlateLookupInput(qimen, qimenManualOverride)
-> getQimenPlate({
     dunType: effective.dunType,
     ju: effective.ju,
     hourPillar: effective.hourPillar
   })
-> render 左側 auto summary
-> render 右側 usage + fallback
```

補充：

* formatter throw 時不呼叫 plate lookup。
* `hourPillar` 永遠來自 formatter。
* manual override 只改 `dunType` / `ju`。
* plate lookup 不回寫 formatter result。
* UI 不顯示 lookup metadata。
* UI 不顯示 cache stats。
* UI 不儲存 manual 狀態到 localStorage / URL。

## 7. 設計邊界

* 奇門 UI 不混入金函玉鏡。
* 左側 summary 顯示 formatter auto result。
* 右側 plate panel 顯示 plate lookup / manual controls。
* 手動覆寫只影響 effective `dunType` / `ju`。
* `hourPillar` 永遠使用 formatter result。
* 手動覆寫不改 formatter result。
* 手動覆寫不是 method switch。
* `nullPlate` 是資料未完成，不是錯誤。
* 缺盤 fallback 不應阻止左側定局摘要顯示。
* plate lookup 不重新計算定局。
* UI 不預設顯示 debug metadata。
* UI 不顯示 cache stats。
