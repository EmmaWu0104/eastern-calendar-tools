# 金函玉鏡日盤人工驗證清單

## 一、目前完成範圍

目前金函玉鏡日盤已完成下列範圍：

- 已整理金函玉鏡 60 甲子日 x 陽遁 / 陰遁，共 120 盤資料。
- 每個日柱皆包含黑黃道十二時辰資料。
- `src/jinhanYujing.js` 已提供純查表函式。
- 主畫面已接入金函玉鏡日盤 UI。
- 目前陰陽遁採手動選擇。
- 目前尚未實作超神接氣自動判斷。
- 目前不會自動依冬至 / 夏至切換陰陽遁。

## 二、驗證前提

- 日柱由既有干支曆核心提供。
- 金函玉鏡查表 key 為 `dayPillar + dunType`。
- `dunType` 目前由畫面手動選擇。
- 青龍黑黃道只依日柱顯示，與金函玉鏡陰陽遁盤本身不連動。
- 喜神、財神、陰貴神、陽貴神目前讀自金函玉鏡資料 `meta`。
- 使用手錶時間 / 瀏覽器本機時間。
- 尚未套用真太陽時。

## 三、固定人工驗證案例

| 案例 | 輸入 / 操作 | 驗證重點 |
| --- | --- | --- |
| 1. 甲子日 + 陽遁 | 讓查詢結果落在甲子日，陰陽遁選擇「陽遁」 | 金函玉鏡盤顯示「陽遁甲子日」；中宮顯示來源資料中的 `center`；九宮格八宮完整；艮宮應同時顯示喜、財、陰；坤宮應顯示陽；青龍黑黃道應有 12 筆；第一筆時段應為 `23 ~ 01`；第一筆時柱應為甲子；第一筆黑黃道神應為金匱；第一筆吉凶應為黃道。 |
| 2. 甲子日 + 陰遁 | 同一甲子日，手動切換為「陰遁」 | 盤名改為「陰遁甲子日」；日柱仍為甲子日；青龍黑黃道仍為甲子日的 12 筆；喜財陰陽 chip 仍依該盤 `meta` 顯示；切換時畫面不應出錯。 |
| 3. 乙丑日 + 陽遁 / 陰遁 | 讓查詢結果落在乙丑日，分別切換陽遁與陰遁 | 陽遁可查到「陽遁乙丑日」；陰遁可查到「陰遁乙丑日」；九星 badge 正常顯示；八門只顯示單字，不顯示「門」；宮位洛書數在左下角；方位在右下角。 |
| 4. 丁卯日 + 陽遁 | 讓查詢結果落在丁卯日，陰陽遁選擇「陽遁」 | 離宮顯示喜；兌宮顯示財、陰；乾宮顯示陽；同宮多個 chip 垂直排列，不覆蓋九星與八門。 |
| 5. 手機版 / 窄螢幕 | 將瀏覽器寬度縮到 760px 以下 | 金函玉鏡盤與青龍黑黃道改回上下排列；九宮格不爆版；青龍黑黃道表不撐爆整頁；若表格過窄，允許表格區水平捲動。 |
| 6. 錯誤 / 空資料保護 | 模擬尚無日柱或查無資料狀態 | 若沒有日柱，應顯示「尚無日柱資料，無法顯示金函玉鏡日盤」；若查無資料，應顯示「查無金函玉鏡日盤資料」；不應造成整頁 JavaScript error。 |

## 四、回歸檢查清單

每次修改金函玉鏡相關功能後，應確認：

- 執行 `npm test`。
- 執行 `git diff --check`。
- 執行 `node --check src/main.js`。
- 執行 `node --check src/jinhanYujing.js`。
- 本機頁面可載入。
- 陽遁 / 陰遁切換正常。
- 青龍黑黃道顯示 12 筆。
- 不修改 `data/solar_terms_1899_2101.json`。
- 不修改已確認的金函玉鏡資料，除非有明確資料修正原因。
