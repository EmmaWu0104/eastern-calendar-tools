# 日本版七十二候候名資料來源整理

## 一、資料來源原則

- 日本版七十二候候名需採可靠來源。
- 第一優先來源建議使用日本國立天文台「暦Wiki / 七十二候」。
- 可使用其他日本曆法整理頁交叉確認，例如「暦生活」等。
- 本包只整理候名清單與來源，不做規則改動。
- 若不同來源候名寫法不同，先記錄差異，不自行武斷合併。
- 若有新字體 / 舊字體差異，第一版先保留主要來源寫法，必要時在 `notes` 欄註記。
- 假名讀音第一版先不做，可後續再加 `kana` 欄位。

本文件採用的資料來源：

- 主要來源：国立天文台 暦計算室「暦Wiki / 季節 / 七十二候」  
  https://eco.mtk.nao.ac.jp/koyomi/wiki/B5A8C0E12FBCB7BDBDC6F3B8F5.html
- 交叉確認：暦生活「七十二候一覧」春 / 夏 / 秋 / 冬  
  https://www.543life.com/content/about72seasons-sp/  
  https://www.543life.com/content/about72seasons-su/  
  https://www.543life.com/content/about72seasons-au/  
  https://www.543life.com/content/about72seasons-wi/

日本版候名以国立天文台 暦Wiki 表格中的「宝暦暦以降」欄為主。暦Wiki 同頁也列出「宣明暦以前」「貞享暦」與讀音；本階段不取讀音，只整理候名。

## 二、既有中國版資料對照

本包已閱讀目前專案的 `data/seventy_two_hou.json`。現有資料結構為：

- 最外層以節氣名稱作為 key。
- 每個節氣下有 3 筆候資料。
- 每筆候資料目前包含：
  - `houIndex`
  - `globalHouIndex`
  - `phase`
  - `name`
  - `shortName`
  - `description`

本文件表格沿用既有中國版資料的欄位順序與索引：

- 節氣名稱
- `houIndex`
- `globalHouIndex`
- `phase`
- `name`
- `shortName`

既有資料檢查結果：

- 24 節氣皆存在。
- 每節氣 3 候。
- 全年共 72 候。
- `globalHouIndex` 為 1～72，不重複、不缺漏。
- 本包不要更動既有中國版資料。

注意：日本來源常用 `啓蟄`、`処暑` 等日本字形；本文件表格的「節氣」欄沿用專案既有中國版節氣 key，例如 `驚蟄`、`處暑`，避免影響後續資料對齊。

## 四、資料對齊原則

- 日本版候名必須依同一個 `globalHouIndex` 對齊中國版候名。
- 不因候名不同而改變候序。
- 不因日本版候名不同而新增或刪除候。
- 每個節氣仍固定 3 候。
- 中日版共用同一套時間位置。
- 七十二候時間判斷邏輯完全不變。

## 五、候名處理原則

- 日本版候名第一版以漢字候名為主。
- 不加入日文解釋長文。
- 不加入假名讀音。
- 不加入日期範圍。
- 不加入日本節氣日期。
- 不加入地區判斷。
- 若日本版候名中出現和中國版相同者，也照實填入。
- 若來源出現「候名」與「読み」分開，本包只取候名。
- 若候名有簡繁 / 新舊字體差異，先以主要來源原文為準，並在 `notes` 註記。
