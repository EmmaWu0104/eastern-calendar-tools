# 九宮飛星交接摘要

本文件整理九宮飛星第一階段至第七包完成後的功能狀態、規則、測試與後續待辦。現階段只做時間飛星，不做陽宅飛星。

## 功能範圍

目前支援：

- 運盤。
- 年盤。
- 月盤。
- 日盤。
- 時盤。
- 五盤整合顯示。
- UI 顯示九宮格。
- `starDisplayName` 圓圈數字顯示名稱，例如 `① 白貪狼`。
- 九宮格 UI 採中央星名、左下宮位洛書、右下方位名稱的格內版型。

明確排除：

- 陽宅飛星。
- 山星、向星。
- 宅運盤、坐向盤。
- 替卦、兼線、元龍。
- 吉凶、旺衰、斷語。
- 真太陽時。

## 時間規則

- 第一版使用手錶時間 / 瀏覽器本機時間。
- 第一版未套用真太陽時。
- 年盤沿用干支曆核心，以立春切換有效年份。
- 月盤沿用干支曆核心，以 12 節換月。
- 日盤使用 23:00 換日後的日柱。
- 時盤使用 23:00 換日後的日支與目前時支。
- 運盤依輸入時間的西元年份與三元九運判斷，不依立春切換。

## 五盤規則摘要

### 運盤

- 依三元九運。
- 幾運即幾星入中。
- 一律順飛。
- `2024-2043` 為九運，九入中宮。

### 年盤

- 使用立春換年後的有效年份。
- 年星入中公式：`11 - 西元年份各位數字和`，結果 normalize 到 `1-9`。
- 一律順飛。
- `2026` 丙午年為一白入中。

### 月盤

- 依年支三組與月支查表。
- 年支三組為 `子午卯酉`、`辰戌丑未`、`寅申巳亥`。
- 一律順飛。
- `2026` 丙午年巳月為五入中。

### 日盤

- 依 24 節氣分六組。
- 六組為 `冬至順`、`雨水順`、`穀雨順`、`夏至逆`、`處暑逆`、`霜降逆`。
- 依六十甲子序號 `mod 9` 查各組 `centers`。
- 順逆方向由節氣系統決定。

### 時盤

- 依目前節氣使用 `winterForward` 或 `summerReverse`。
- 使用 `dayBranch + hourBranch` 完整查表。
- 不用公式推導，避免流派差異與表格轉置風險。

## 核心檔案

- `src/flyingStars.js`：九宮飛星核心，包含飛佈、五盤計算、月盤表、日盤表與時盤表。
- `src/main.js`：UI 呼叫五盤整合函式並 render 九宮飛星結果。
- `styles/main.css`：九宮飛星 UI 樣式。
- `tests/flying-stars-testcases.json`：九宮飛星測試資料。
- `tests/run-tests.js`：測試 runner，固定測試基準時區並執行干支曆與九宮飛星測試。
- `docs/03_九宮飛星規則設計草案.md`：規則設計。
- `docs/04_九宮飛星資料結構與測試計畫.md`：資料結構與測試計畫。
- `docs/05_九宮飛星流派差異與待確認事項.md`：流派差異與待確認事項。
- `docs/06_九宮飛星交接摘要.md`：本交接摘要。

## UI 格內版型

九宮格每格顯示採用：

- 正中央：九宮飛星顯示名稱，例如 `⑨ 紫右弼`。
- 左下角：宮位名稱與洛書數，例如 `巽4`。
- 右下角：方位名稱，例如 `東南`。

方位名稱是顯示層 label，依宮位 id 對應固定文字；此版型調整未影響九宮飛星計算規則。

## 測試狀態

目前測試重點：

- `flyStars` 順飛 / 逆飛。
- 運盤九運。
- 年盤 `2026` 一白入中。
- 月盤三組年支。
- 日盤六組節氣系統。
- 時盤 `winterForward` / `summerReverse` 完整查表案例。
- 23:00 換日。
- 夏至前後順逆切換。
- 五盤整合。

目前測試結果：

- `npm test` 通過，九宮飛星 `60 cases`。
- `TZ=UTC npm test` 通過，九宮飛星 `60 cases`。
- 測試 runner 固定 `Asia/Taipei` 作為測試基準。這是目前測試案例的手錶時間基準，不代表產品邏輯使用 UTC。

## 人工驗證建議案例

| 輸入時間 | 驗證重點 |
| --- | --- |
| `2026-05-29T09:30` | 運盤九入中；年盤一入中；月盤五入中；日盤七入中；時盤六入中 |
| `2026-05-29T22:59` | 日盤 basis：癸卯；時盤 basis：癸卯、亥時 |
| `2026-05-29T23:00` | 日盤 basis：甲辰；時盤 basis：甲辰、子時 |
| `2026` 夏至前後 | 夏至前日盤 / 時盤順飛；夏至後日盤 / 時盤逆飛 |

## 後續待辦

- 未來可做真太陽時，但需等核心穩定後再做，且會影響日柱、時柱、節氣交界判斷。
- 若要處理陽宅飛星，必須另開模組，不要混入目前時間飛星。
- 若要加入吉凶斷語，需另建資料來源與流派規則，不要直接寫死在目前核心。
- 可考慮增加匯出 JSON / 複製盤面功能。
- 可考慮加入人工驗證快速按鈕，但不要影響核心。
